常见问题：框架启动报错

本页列出 GoBlade 启动阶段常见报错及解决方案，帮助你快速定位问题。
❌ 端口被占用

报错信息示例：

listen tcp :8080: bind: address already in use

原因： 当前端口（如 8080）已被其他程序占用。

解决方式：

  • ① 更改 config.yaml 中的 port 参数，例如换成 8090
  • ② 释放占用端口（如杀掉占用 8080 的进程）
# Linux/macOS 查看占用
lsof -i :8080

# Windows 查看端口占用
netstat -ano | findstr 8080
taskkill /F /PID 进程号

📦 其他可能的问题
  • 配置文件格式错误： 请检查 config.yaml 是否有缩进或冒号错误
  • 数据库连接失败： 检查 mysql.host、user、password 是否正确
  • 未安装依赖包： 执行 go mod tidy 同步依赖
  • main.go 没有执行： 请从项目根目录运行 go run .
