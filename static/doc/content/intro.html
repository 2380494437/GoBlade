GoBlade 框架介绍

轻量、清晰、可拓展的 Go 后端模板框架
👤 作者 菜鸟八哥
🛠 技术栈 Go + Gin + MySQL + Layui
🎯 项目定位 简洁、高性能，适合中小型接口服务
👥 适用人群 Go 初学者、ThinkPHP 转 Go 用户、后端开发者
📂 支持功能 多版本路由、自动建表、安装引导界面、自定义错误页、静态资源托管、中间件机制、IP黑名单、内存控制、并发限流、CORS控制、Gzip压缩、维护模式等
📈 项目优势
  • 轻量无依赖：除 Gin 外几乎无三方依赖，部署方便
  • 结构清晰：分层明确，控制器、路由、配置独立管理
  • 多版本接口：支持 /api/v1 与 /api/v2 等版本分离，迭代无忧
  • 自动建表：安装时可自动生成数据表结构
  • 可拓展性高：支持集成 GORM、JWT、RBAC 权限等
✅ 推荐使用场景
  • 🎯 Go 初学者练习：快速理解完整后端架构
  • 💡 中小项目起步：接口清晰，适合初期业务搭建
  • 🚀 后台管理/系统中台：结合前端快速开发管理系统
  • 📦 个人/团队脚手架：可做内部项目标准模板
📊 同类框架对比
对比项 GoBlade（Go） ThinkPHP（PHP） Spring Boot（Java）
语言运行模式 编译执行，常驻内存 解释执行，每次请求重启 编译执行，JVM 长驻
并发性能 ⭐⭐⭐⭐⭐（极高） ⭐⭐（依赖 FPM） ⭐⭐⭐（适中）
部署复杂度 ⭐⭐⭐（需构建+守护） ⭐⭐（Nginx + PHP 即可） ⭐⭐⭐⭐（需 JDK、打包、服务）
框架体积 小（单文件可运行） 中 大（Jar包 + 依赖）
学习曲线 ⭐⭐⭐（语法简单） ⭐⭐（门槛低） ⭐⭐⭐⭐（Java 背景要求）
扩展生态 成熟生态（Gin/GORM/logrus） 丰富（大量类库 + 模板） 强大（Spring Cloud 微服务）
