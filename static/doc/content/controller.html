可以修改的文件和开发入口说明

本页用于帮助开发者快速识别哪些目录和文件可以修改，哪些属于框架核心不建议动。
✅ 推荐修改区域（你的业务代码）
  • /controllers/：你的接口逻辑写在这里，按版本划分
  • /routes/：注册你的路由路径，如 /api/v1/hello
  • /utils/：可以扩展返回结构、JSON工具等，把你开发的一些通用工具模块丢到这个目录，自己用到的时候按需加载
  • /templates/：页面渲染模板（如 HTML 错误页,一般不需要修改，除非你想要自己DIY，为什么放到这里而不放到/static/？因为/static/里是资源是可以直接被公网访问到的，而这里的不能，确保原始模板代码的安全）
  • /config.yaml：配置端口、模式、并发限制、数据库等
  • /static/：静态文件目录（也就是你的项目前端目录,所有的html，js，css，图片等整个前端代码都放到这个目录,丢进去就可以直接访问到）
  • /middlewares/auth.go：权限拦截器（可按需改,比如你的接口需要用到用户鉴权，则在这里统一开发）
⛔ 框架核心区域（不建议改动）
  • /middlewares/core/：内置中间件，如 Gzip、限流、CORS 等
  • /config/：配置加载器、数据库连接（框架自动调用）
  • /main.go：仅用于启动，不推荐放业务代码

⚠️ 修改这些文件可能影响框架稳定性，建议仅在非常熟悉后再修改。

🧩 控制器文件结构示例 
// controllers/v1/hello.go

package v1

import (
    "github.com/gin-gonic/gin"
    "GoBlade/utils"
)

func HelloHandler(c *gin.Context) {
    utils.JSON(c, 200, "你好，GoBlade！", gin.H{
        "path": c.Request.URL.Path,
    })
}
📂 控制器文件命名建议
  • 按模块拆分，例如：user.go、article.go
  • 每个版本放在对应目录下，如：controllers/v1/
  • 函数以 xxxHandler 结尾，如：LoginHandler
