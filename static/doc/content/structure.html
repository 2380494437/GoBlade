框架目录结构说明

GoBlade 框架采用模块化设计，以下是项目完整结构与说明。
  • /go/
    • main.go - 启动入口，初始化配置、路由、注册中间件
    • config/
      • blacklist.txt - IP黑名单文件
      • project_config.go - 项目静态配置结构定义
      • loader.go - 加载 config.yaml，支持热重载
      • db.go - 初始化全局数据库连接池 DB
    • controllers/
      • v1/
        • install.go - 安装逻辑，自动建表示例
      • v2/
        • install.go - v2 安装逻辑（可扩展）
    • routes/
      • router.go - 路由注册入口，挂载 v1/v2 路由
      • v1/v1.go - v1 接口定义与注册
      • v2/v2.go - v2 接口定义与注册
    • middlewares/
      • auth.go - 自定义权限认证中间件
      • core/
        • body_limet.go - 上传大小限制中间件
        • cors.go - 跨域处理
        • domain_check.go - 域名白名单验证
        • forbid.go - 请求路径拦截
        • gzip.go - gzip 响应压缩
        • ip_blacklist.go - 黑名单 IP 拦截
        • ip_limit.go - 每 IP 并发限制
        • limit.go - 全局并发限制
        • maintenance.go - 全站维护模式
        • memory_guard.go - 内存占用限制
        • recovery.go - panic 捕获与错误页渲染
        • timeout.go - 请求超时控制
    • static/
      • index.html - 首页或入口页
      • doc/index.html - 📘 开发文档主页面
      • layui/ - 前端 UI 框架资源（Layui）
    • templates/
      • error.html - 通用错误页
      • 403.html / 404.html / 500.html 等 - 对应错误状态页
    • utils/
      • error.page.go - 错误页渲染工具
      • json.go - JSON 编解码处理
      • response.go - 统一 JSON 返回封装
    • config.yaml - 主配置文件（支持热重载）
    • go.mod / go.sum - Go 项目依赖声明文件
    • rsrc_windows_amd64.syso - Windows 图标资源文件
✅ 建议：保持目录结构清晰可维护，配置与代码逻辑严格分离。
