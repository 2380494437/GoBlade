Windows 部署：常见坑合集

以下是 GoBlade 在 Windows 下运行、部署中初学者常见问题及对应解决方法。
❌ 1️⃣ 控制台启动后被关闭

原因： 双击运行程序会自动关闭窗口

建议： 使用命令行运行程序：

cmd 或 PowerShell：
go run main.go
goblade.exe
🌐 2️⃣ 防火墙拦截端口

现象： 浏览器无法访问 http://localhost:8080

解决：

控制面板 → 系统和安全 → Windows 防火墙 →
高级设置 → 入站规则 → 新建规则 →
选择端口 → 输入 8080 → 允许连接
📁 3️⃣ 路径分隔符错误

配置中路径建议使用 / 而不是 \\

错误： .\config\\file ❌

正确： ./config/file ✅

🧩 4️⃣ config.yaml 修改不生效

可能原因：

  • 配置文件编码为 UTF-16 而非 UTF-8
  • 未保存成功（被系统锁定）

建议： 使用 VSCode 或 Notepad++ 编辑保存为 UTF-8

🔍 5️⃣ 没有安装 Go 或环境变量未配置

症状： 执行 go run 报错

解决：

  • 确保已安装 Go 并配置 PATH 环境变量
  • 在命令行中执行 go version 验证
✅ Windows 不适合生产部署，建议部署到 Linux 服务器，并使用反向代理 + 守护进程管理。
