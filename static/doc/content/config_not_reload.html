常见问题：修改 config.yaml 配置不生效？

GoBlade 框架支持配置热重载，但⚠️并不是所有配置都能在运行时立即生效。
✅ 支持热重载的配置项（无需重启）
  • max_concurrent_requests：全局并发限制
  • max_body_size：上传大小限制
  • maintenance_mode：维护模式开关
  • ip_blacklist_file：黑名单路径
  • cors 跨域配置
  • forbidden_paths / dirs / suffixes
  • allowed_hosts 域名白名单
  • mysql_enabled：是否启用数据库,注意，数据库的修改操作会导致数据库连接池实时重置,一般不要改
  • table_prefix：表名前缀（影响自动建表）
  • mysql 下所有字段：host、port、user、password、dbname 等

说明：这些配置由热加载模块定期轮询并刷新内存，无需重启。(此处未一一列举)

❌ 修改后必须重启才能生效
  • port：监听端口
  • bind_ip：监听地址
  • static_dir：静态资源目录
  • error_dir：错误页模板目录
  • default_document：默认首页文档

说明：这些配置会在程序初始化时加载，并不支持运行中热替换。

🧪 判断配置是否生效的方法
  • 查看控制台日志是否输出“配置文件加载完成”
  • 可通过调用接口打印 config.Config 内容来验证
  • 重启程序后再观察端口变化、静态目录是否更新
✅ 建议：调试前先确认你改的是支持热重载的配置项，否则必须重启程序。
